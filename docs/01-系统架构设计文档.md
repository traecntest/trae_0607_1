# 低空经济与无人机交通管理系统（UTM）- 系统架构设计文档

## 1. 系统概述

### 1.1 项目背景
随着低空经济的快速发展，无人机在物流配送、巡检监控、应急救援等领域得到广泛应用。无人机交通管理系统（UTM - Unmanned Aircraft System Traffic Management）旨在为低空飞行的无人机提供安全、高效的空中交通管理服务。

### 1.2 设计目标
- **高吞吐量**：支持数百架无人机同时接入，处理高频遥测数据
- **低延迟**：实时监控与冲突检测，确保飞行安全
- **高可用**：微服务架构，支持水平扩展和故障隔离
- **可扩展**：模块化设计，便于功能扩展和技术升级

### 1.3 技术栈
- **开发框架**：.NET 8.0 / ASP.NET Core Web API
- **开发工具**：Visual Studio 2022
- **微服务架构**：事件驱动 + 领域事件
- **高性能计算**：System.Numerics.Vectors (SIMD)、System.IO.Pipelines
- **异步流处理**：System.Threading.Channels、IAsyncEnumerable
- **测试框架**：xUnit、Moq、FluentAssertions、BenchmarkDotNet
- **容器化**：Docker / Testcontainers

## 2. 总体架构设计

### 2.1 架构风格
采用**微服务 + 事件驱动**的混合架构：
- 微服务实现业务功能的解耦和独立部署
- 事件驱动实现服务间的松耦合通信
- 内存事件总线实现高性能实时数据交换

## 3. 微服务划分与职责

### 3.1 服务列表

| 服务名称 | 项目名称 | 职责描述 |
|---------|---------|---------|
| API网关 | UTM.ApiGateway | 统一入口、REST API、Swagger文档、聚合服务 |
| 遥测服务 | UTM.TelemetryService | 数据接入、状态管理、轨迹跟踪、模拟数据 |
| 冲突检测服务 | UTM.ConflictDetectionService | 冲突检测、避让规划、严重程度评估 |
| 共享内核 | UTM.SharedKernel | 公共模型、事件契约、计算库、基础设施 |

### 3.2 遥测服务 (Telemetry Service)

#### 核心功能
1. **数据接入网关**
   - 基于 System.IO.Pipelines 的高性能数据接入
   - 支持 JSON 行格式和二进制格式
   - 单实例支持 >10万 消息/秒的吞吐量

2. **背压控制管道**
   - 基于 System.Threading.Channels 的有界队列
   - 生产者-消费者模式
   - 支持异步流 IAsyncEnumerable

3. **无人机状态存储**
   - 内存状态存储
   - 实时位置、速度、电量等状态
   - 支持快速查询

4. **轨迹跟踪服务**
   - 历史轨迹记录（滑动时间窗）
   - 轨迹预测（基于当前速度）
   - 飞行计划匹配

5. **模拟数据生成器**
   - 高保真无人机模拟
   - 支持数百架无人机同时飞行
   - 支持多种飞行模式

### 3.3 冲突检测服务 (Conflict Detection Service)

#### 核心功能
1. **冲突检测算法**
   - TCPA/DCPA 最近会遇时间/距离计算
   - 基于时间窗的滑动检测算法
   - AABB 包围盒碰撞检测

2. **SIMD 加速计算**
   - 使用 System.Numerics.Vectors 向量化
   - 批量距离计算加速
   - 三维空间矩阵运算

3. **避让决策规划**
   - 基于优先级的避让策略
   - 多维度避让建议（高度、航向、速度）
   - 冲突严重程度分级（低/中/高/危急）

4. **冲突事件管理**
   - 活动冲突追踪
   - 冲突事件发布
   - 冲突解决确认

### 3.4 API 网关 (API Gateway)

#### 核心功能
1. **RESTful API 接口**
   - 无人机管理接口
   - 遥测数据查询接口
   - 轨迹查询接口
   - 冲突告警接口
   - 系统状态接口

2. **服务聚合**
   - 聚合多个微服务的数据
   - API 组合与转换
   - 统一数据格式

3. **托管服务**
   - 模拟器托管服务
   - 遥测处理托管服务
   - 冲突检测托管服务

## 5. 核心算法设计

### 5.1 TCPA/DCPA 冲突预测算法

#### 算法原理
TCPA (Time to Closest Point of Approach)：最近会遇时间
DCPA (Distance at Closest Point of Approach)：最近会遇距离

通过计算两个运动物体的相对速度和相对位置，预测它们在未来何时会最接近，以及最近距离是多少。

#### 计算公式
```
相对速度: V_rel = V2 - V1
相对位置: P_rel = P2 - P1
TCPA = - (P_rel · V_rel) / |V_rel|²
DCPA = |P_rel + V_rel * TCPA|
```

#### 判断规则
- TCPA > 0 且 TCPA ≤ 预测时间范围：存在潜在冲突
- DCPA ≤ 安全距离阈值：确认冲突

### 5.2 滑动时间窗检测算法

#### 算法原理
使用滑动时间窗口维护最近一段时间内的遥测数据，对窗口内的所有无人机进行两两冲突检测。

#### 时间窗口配置
- 窗口大小：60秒（可配置）
- 滑动间隔：1秒
- 检测频率：10Hz（可配置）

#### 算法流程
1. 新遥测数据进入窗口
2. 过期数据移出窗口
3. 对窗口内所有无人机进行两两组合
4. 每对无人机执行 TCPA/DCPA 计算
5. 检测到冲突则发布事件

### 5.3 AABB 包围盒碰撞检测

#### 算法原理
使用轴对齐包围盒 (Axis-Aligned Bounding Box) 进行快速碰撞检测，适用于大范围快速筛选。

#### 三维包围盒定义
```
Min: (LonMin, LatMin, AltMin)
Max: (LonMax, LatMax, AltMax)
```

#### 相交判定
两个包围盒相交当且仅当在三个坐标轴上都有重叠：
```
A.Max >= B.Min 且 A.Min <= B.Max  (对每个坐标轴)
```

#### SIMD 加速
使用 `System.Numerics.Vector3` 进行向量化比较，一次运算完成三个坐标轴的比较。

### 5.4 冲突严重程度分级

| 等级 | 触发条件 | 响应时间要求 |
|-----|---------|------------|
| Low (低) | TCPA < 30s 且 DCPA < 0.9×安全距离 | 60秒内响应 |
| Medium (中) | TCPA < 15s 或 DCPA < 0.6×安全距离 | 30秒内响应 |
| High (高) | TCPA < 5s 或 DCPA < 0.3×安全距离 | 10秒内响应 |
| Critical (危急) | 当前距离 < 安全距离 | 立即响应 |

## 6. 性能设计

### 6.1 高吞吐量数据接入

#### System.IO.Pipelines 优势
- 内存池化，减少 GC 压力
- 流水线式处理，提高吞吐量
- 原生支持跨平台

#### 性能指标（单实例）
- 遥测数据接入：> 100,000 消息/秒
- 平均处理延迟：< 1ms
- 内存占用：< 500MB（1000架无人机）

### 6.2 背压控制机制

#### Channel 背压策略
- 有界 Channel，设置容量上限
- 生产者等待模式：`WaitToWriteAsync`
- 消费者异步拉取：`ReadAllAsync`
- 支持取消操作

#### 背压触发条件
- 管道容量达到 80% 时触发告警
- 管道容量达到 100% 时生产者阻塞

### 6.3 SIMD 向量化加速

#### 适用场景
- 批量距离计算
- 批量包围盒检测
- 向量矩阵运算

#### 预期性能提升
- 距离计算：3~5倍加速
- 包围盒检测：4~6倍加速
- 整体冲突检测：2~3倍加速

### 6.4 水平扩展策略

#### 无状态服务扩展
- API 网关：无状态，可直接水平扩展
- 冲突检测服务：可按区域分片扩展

#### 有状态服务扩展
- 遥测服务：按无人机 ID 分片（一致性哈希）
- 状态存储：使用分布式缓存
